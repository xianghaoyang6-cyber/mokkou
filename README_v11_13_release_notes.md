# 木工 PWA v11.13 版本顯示校正與大型展開按鈕穩定加固版

## 版本定位

v11.13 是 v11.12 的小修補版，主要處理：

1. 首頁與頁尾版本顯示修正。
2. 大型展開 / 收合按鈕二次加固。
3. 降低舊版 fold-mark、data-label、CSS 偽元素膠囊再次干擾的機率。
4. 保留所有大型區塊預設收合。

## 本版不改的內容

- 核心櫃體計算
- 盒子計算
- 紙盒刀模幾何
- 旋鈕核心計算
- v10.91 分片拼接孔 / 榫接補強
- v10.90 照片庫 ZIP 備份 / 還原
- v10.89 IndexedDB 照片庫
- 施工圖三視圖
- SVG / DXF / PNG 主流程
- localStorage 主要資料結構

## v11.13 修改重點

### 1. 版本顯示修正

修正 v11.12 檔案中首頁仍顯示「我の木工 PWA v11.11」的問題。

現在顯示為：

- 我の木工 PWA v11.13
- PWA v11.13 FOLD STABLE
- 頁尾顯示 v11.13 版本資訊

### 2. 大型展開 / 收合按鈕加固

v11.13 保留 v11.12 的實體按鈕架構，並新增 v1113 守護邏輯：

- 相容既有 `button.v1112-fold-button`
- 同一顆按鈕同時加上 `v1112-fold-button v1113-fold-button`
- 避免 v11.12 與 v11.13 互相重建按鈕造成重複
- 收合時顯示「展開」
- 展開時顯示「收合」
- 重新渲染、切頁、展開收合、DOM 變動、頁面返回後再次同步

### 3. 降低重刷與互相觸發

v11.13 會接手 v11.12 的 MutationObserver，避免舊 observer 監看 inline style 造成過度重刷。

v11.13 只監看：

- details 的 `open`
- class 變化
- DOM 子節點變化

## GitHub Pages 更新方式

1. 解壓縮 ZIP。
2. 將 ZIP 內的 `index.html` 上傳並覆蓋 GitHub Pages 根目錄的 `index.html`。
3. 一起上傳 `clear-cache_v11_13_fold_button_stability.html`。
4. 手機打開清快取頁。
5. 執行「清除 PWA 快取 / Service Worker」。
6. 回到正式網站重新驗收。

## 手機驗收重點

請優先測：

1. 首頁是否顯示 v11.13。
2. 計算頁大型卡片右側是否有「展開 / 收合」。
3. 切到施工圖、排版圖、參數庫、備份頁後，按鈕文字是否仍存在。
4. 展開後是否變成「收合」。
5. 收合後是否變回「展開」。
6. 小型功能按鈕，例如「計算」、「下載 SVG」、「備份」，不應被影響。

## 設備基準

本專案後續仍以使用者的 10W 藍光雷射切割機為主要基準：

- 工作範圍：300 × 300 mm
- 建議有效切割區：280 × 280 mm
- SVG、紙盒刀模、板材排版、測試片、參數庫與施工圖皆以此為主要參考
